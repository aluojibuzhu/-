## 1. 模块定位

成本看板是 MVP 闭环的第五阶段。项目管理(1)→成本填报(2)→审批入账(3)→监控预警(4)→**成本看板(5)**。

核心目标：**增强项目资金数据的时效性**。通过看板实时展现项目资金状态，通过灵活报表导出支撑汇报和分析。

### 1.1 旧设计的问题

| 问题 | 说明 |
|------|------|
| 职责重叠 | 原5.2"预算执行对比"与模块4看板功能重复 |
| 指标混杂 | 原5.1含"预警状态"，与模块4功能重叠 |
| 无报表能力 | 只展示不导出，无法用于汇报 |
| 产值概念多余 | MVP已移除产值，但原5.2仍含 |

### 1.2 重构方案

从 3 个子模块重构为 **2 个独立功能页面**（5.2 下含 4 个专项报表）：

```
旧: 5.1项目总览 / 5.2预算执行对比 / 5.3盈亏分析
新: 5.1成本总览 / 5.2专项报表（4个独立报表页）
```

## 2. 设计决策

| 决策项 | 决定 | 理由 |
|--------|------|------|
| 服务角色 | 老板+项目经理+成本会计，单一总览页+下钻跳转 | 一个页面覆盖全局，下钻到项目详情 |
| 预警指标 | 不包含预警相关指标 | 职责分离，预警由模块4负责 |
| 顶部指标 | 6个纯资金指标（预算/已执行/余额/合同/利润空间/本月入账） | 聚焦资金状态+时效变动 |
| 项目列表 | 三状态切换（进行中/已完工/全部）+ 搜索 | 快速定位 |
| 项目下钻 | 点击行跳转项目详情页（模块1.3） | 复用已有页面 |
| 报表结构 | 4个独立报表页，各自固定列+合理筛选 | 不同侧重点，筛选在上表格在下 |
| 筛选布局 | 筛选条件在上方，表格在下方 | 与成本总览一致 |
| 导出格式 | 仅 Excel .xlsx | MVP阶段最常用 |
| 利润空间 | 合同金额 − 预算总额 | 预期利润空间，非实际盈亏 |

```
5. 成本看板
```

## 4. 页面设计

### 4.1 成本总览 `5.1`

```
┌─ 页面标题 + [刷新] ─────────────────────────────────────┐
├─ 6个资金指标横排 ───────────────────────────────────────┤
│  预算总额 | 已执行成本 | 预算余额 | 合同金额 | 利润空间 | 本月入账+环比 │
├─ 状态切换tab: [进行中 18] [已完工 7] [全部 25] ────────┤
├─ 搜索栏: [项目名称/编号]    共 N 个项目 ────────────────┤
├─ 项目资金表格 ─────────────────────────────────────────┤
│  编号 | 名称 | 合同金额 | 预算 | 已执行 | 余额 |        │
│  利润空间(+/-) | 本月入账 | 执行率(进度条)             │
│  点击行 → 跳转项目详情页（模块1.3）                     │
└────────────────────────────────────────────────────────┘
```

**指标计算**：
- 预算总额/已执行/余额/合同金额/利润空间：按当前tab筛选的项目范围合计
- 本月入账：当月 cost_posting_record 金额合计
- 环比：与上月同期比较的百分比变化

### 4.2 专项报表 `5.2`

4个独立报表页面，每个：**筛选在上方，表格在下方，底部导出按钮**。

#### 5.2.1 项目资金汇总表

**侧重点**：项目维度资金概览，适合老板和项目经理。

**筛选**（上方）：项目（多选）+ 时间范围（月/季度/自定义）
**表格**（下方）：项目编号/名称/合同金额/预算总额/已执行/余额/利润空间/执行率

#### 5.2.2 科目成本明细表

**侧重点**：成本科目维度的预算执行，适合成本会计。

**筛选**（上方）：项目（多选）+ 科目（多选）+ 时间范围
**表格**（下方）：项目名称/一级科目/二级科目/预算金额/已执行金额/余额/执行率

#### 5.2.3 入账流水明细表

**侧重点**：逐笔入账跟踪，适合审计和成本会计。

**筛选**（上方）：项目（多选）+ 单据类型（工时/报销）+ 时间范围 + 审批人
**表格**（下方）：编号/项目名称/单据类型/科目/金额/入账日期/审批人

#### 5.2.4 节点预算执行表

**侧重点**：WBS节点维度的预算执行，适合项目经理。

**筛选**（上方）：项目（多选）+ 节点（多选）+ 时间范围
**表格**（下方）：项目名称/节点编号/节点名称/节点预算/已执行/余额/执行率/计划完成日期

每个报表均有独立的 `[导出Excel]` 按钮，导出全量数据。

## 5. 数据源

本模块无需新建数据库表，所有数据从现有表聚合：

| 数据 | 来源表 |
|------|--------|
| 项目预算/已执行/余额 | `proj_info` (total_budget, actual_cost, budget_balance) |
| 合同金额 | `proj_info` (contract_amount) |
| 项目状态 | `proj_info` (status) |
| 入账明细 | `cost_posting_record` (amount, bill_type, create_time, proj_id, node_id, category_id) |
| 成本科目 | `sys_cost_category` |
| WBS节点 | `proj_wbs_node` |
| 审批人 | `cost_posting_record` (post_by) |

## 6. 后端设计

### 6.1 文件清单

```
ruoyi-admin/src/main/java/com/ruoyi/web/controller/project/
├── CostDashboardController.java    # 5.1 成本总览API
└── CostReportController.java       # 5.2 报表导出API

ruoyi-system/src/main/java/com/ruoyi/system/service/project/
├── ICostDashboardService.java + impl
└── ICostReportService.java + impl
```

### 6.2 API

**CostDashboardController** (`/cost/dashboard`):
- `GET /summary?status=` — 6指标汇总数据（按项目状态过滤）
- `GET /list?status=&keyword=&pageNum=&pageSize=` — 项目资金列表（分页+搜索+状态切换）

**CostReportController** (`/cost/report`):
- `POST /preview` — 根据筛选条件返回预览数据（前100行）
- `POST /export` — 生成并返回 .xlsx 文件

### 6.3 指标计算SQL逻辑

```sql
-- 本月入账
SELECT COALESCE(SUM(amount), 0) FROM cost_posting_record
WHERE DATE_FORMAT(create_time, '%Y-%m') = DATE_FORMAT(NOW(), '%Y-%m')

-- 环比
比较本月入账与上月入账的百分比变化
```
