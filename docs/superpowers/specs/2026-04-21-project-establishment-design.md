## 1. 模块定位

项目立项是工程项目成本管理的起点。在此阶段，项目经理需完成完整的项目方案制定——包括基本信息、WBS进度节点、成本预算——经管理员审批通过后，项目正式生效并进入执行阶段。

> 设计原则：立项即做方案，一次审批通过，减少返工。

---

## 2. 页面结构

立项页面为**单页式完整方案**，包含四个区块：

```
新建项目
├── 基本信息区
├── WBS节点-成本分配表格（节点为主视角，各节点填写各成本科目分配金额）
└── 操作区（保存草稿 / 提交审批）
```

---

## 3. 基本信息区

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目编号 | 文本（只读） | — | 系统自动生成，格式：PRJ-YYYYMMDD-序号 |
| 项目名称 | 文本 | 是 | 项目全称，最大50字 |
| 关联客户 | 下拉选择 / 新建 | 是 | 从客户列表选择，也可点击新建客户 |
| 预计开工日期 | 日期 | 是 | 格式：YYYY-MM-DD |
| 预计竣工日期 | 日期 | 是 | 必须晚于开工日期 |
| 预计合同金额 | 数字 | 是 | 单位：元，保留两位小数 |
| 项目简介 | 文本 | 否 | 最大500字 |

**交互说明：**
- 关联客户支持两种方式：
  1. 从下拉列表选择已有客户
  2. 点击"新建客户"按钮，在弹窗中录入客户信息后自动回填
- 新建客户由项目经理操作，无需审批

---

## 4. WBS节点区

### 4.1 节点字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 节点编号 | 文本（只读） | — | 系统自动生成，格式：ND-001，不可编辑 |
| 节点名称 | 文本 | 是 | 如：基础工程、主体结构，最大30字 |
| 预计完成日期 | 日期 | 是 | 格式：YYYY-MM-DD |
| 成本科目分配 | 数字 | 是 | 各成本科目分配到该节点的金额（手动填写） |
| 节点预算 | 数字（计算） | — | 自动累计 = 各成本科目分配之和，不可编辑 |

### 4.2 交互规则

- 初始为空，点击"新增节点"按钮添加节点
- 每行节点可编辑，可删除
- 预计完成日期建议在开工和竣工日期范围内

### 4.3 数据结构

节点为扁平列表，无父子层级关系。节点预算不单独填写，而是在成本预算分配表格中自动累计得出。

---

## 5. WBS节点-成本分配表格

### 5.1 成本科目

成本科目为企业级配置，支持二级科目，企业管理员可自定义添加/修改。

**预置一级科目：**
- 人工费
- 材料费
- 设备费
- 分包费
- 现场杂费
- 管理费
- 其他

**示例二级科目：**

| 一级科目 | 二级科目 |
|----------|----------|
| 人工费 | 项目管理人工、设计人员人工、施工人员人工 |
| 材料费 | 主材、辅材、周转材料 |
| 设备费 | 自有设备折旧、租赁设备 |
| 分包费 | 专业分包、劳务分包 |
| ... | ... |

### 5.2 节点-科目分配表格

**结构：**
横向（行）为 WBS 节点，纵向（列）为节点属性 + 各成本科目分配金额。

```
节点编号 | 节点名称 | 预计完成日期 | 人工费 | 材料费 | 设备费 | 分包费 | ... | 节点预算
ND-001   | 基础工程 | 2026-06-01  | 20万   | 25万   |  5万   |        |      | 50万（自动累计）
ND-002   | 主体结构 | 2026-09-01  | 30万   | 40万   | 10万   |        |      | 80万（自动累计）
```

**字段说明：**
- 节点编号：系统自动生成（ND-001...），不可编辑，首列
- 节点名称：可编辑
- 预计完成日期：可编辑
- 人工费、材料费、设备费、分包费等：各成本科目分配到该节点的金额，由项目经理手动填写
- 节点预算：自动累计 = 该行所有成本科目分配金额之和，不可手动编辑

**交互规则：**
- 初始为空，点击"新增节点"按钮添加节点行
- 每行节点可编辑，可删除
- 各成本科目分配金额由项目经理手动填写
- 节点预算由系统自动计算得出，不可手动修改

### 5.3 校验规则

- 各成本科目分配到节点的金额由项目经理手动填写
- 节点预算 = 该节点各成本科目分配金额之和（系统自动计算）
- 各节点预算之和即为项目总预算
- 提交前必须完成所有节点和成本科目的填写

## 6. 项目状态

| 状态 | 说明 |
|------|------|
| 草稿 | 项目信息未提交，可继续编辑 |
| 审批中 | 已提交，等待管理员审批 |
| 已立项 | 审批通过，项目正式生效 |
| 已驳回 | 审批未通过，退回修改后可重新提交 |
| 进行中 | 项目开工后的状态 |
| 已完工 | 项目竣工后的状态 |

**状态流转：**

```
草稿 → 提交审批 → 审批中 → 通过 → 已立项
                              → 驳回 → 草稿（修改后重新提交）
已立项 → 开工 → 进行中 → 完工 → 已完工
```

## 7. 审批流程

> **设计更新（2026-04-30）**：审批入口统一收归模块3「审批入账」。立项模块只负责「提交审批」操作，审批人（管理员）通过模块3的统一待审批列表进入项目详情完成审批。后端审批 API（PUT /approve, PUT /reject）仍保留在 ProjInfoController 中，由模块3的审批详情页调用。

### 7.1 流程说明

```
项目经理填写完整方案 → 保存草稿 → 确认无误后提交审批
                                 ↓
                        系统校验（预算 vs 节点一致性）
                                 ↓
                        进入模块3「待审批列表」
                                 ↓
                     管理员在模块3查看完整方案
                                 ↓
                     通过 / 驳回（填写驳回原因）
                                 ↓
              通过 → 项目状态变更为"已立项"
              驳回 → 退回项目经理修改
```

### 7.2 审批内容

管理员在模块3审批时查看完整立项方案：
- 基本信息（项目名称、客户、合同金额、日期等）
- WBS节点列表（节点编号、节点名称、预计完成日期、各科目分配金额、节点预算）
- 成本预算分配表格

### 7.3 审批操作

- **通过**：项目状态变更为"已立项"，发送通知给项目经理
- **驳回**：填写驳回原因，项目状态退回"已驳回"，通知项目经理修改
- 审批操作通过模块3的统一审批入口执行，后端 API 仍由 ProjInfoController 提供

## 9. 与原PRD的差异

| 方面 | 原PRD | 本设计 |
|------|-------|--------|
| 立项内容 | 基本信息，审批后才做预算和WBS | 基本信息 + WBS + 成本预算完整方案 |
| WBS设置 | 预算编制阶段 | 立项阶段同步完成 |
| 成本预算 | 预算编制阶段 | 立项阶段同步完成 |
| 成本科目 | 仅一级7大类 | 支持二级科目，企业可配置 |
| 审批角色 | 企业主 | 管理员 |
| 产值概念 | 有 | MVP阶段不需要，已移除 |
| 预算-节点关系 | 成本科目分配到节点 | 节点为主视角，各节点填写各科目分配金额 |
| 节点预算 | 手动填写 | 自动累计各科目分配之和 |
