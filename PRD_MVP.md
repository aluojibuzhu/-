| 版本号 | 日期 | 作者 | 说明 |
|--------|------|------|------|

## 1. 产品概述

### 1.1 产品定位

面向**中小型建筑企业**的**SaaS 工程项目成本管理平台**，解决企业项目成本管理中"预算执行失控"和"成本进度脱节"两大核心痛点。

### 1.2 MVP核心闭环

```
立项审批 → 预算编制 → 成本填报 → 审批入账 → 预警监控 → 成本看板
    ↑                                                           │
    └───────────────────────────────────────────────────────────┘
```

### 1.3 目标用户与核心诉求

| 角色 | 核心诉求 | 在MVP中的职责 |
|------|----------|--------------|
| **企业主/老板** | 掌控全局经营风险 | 审批项目立项、查看成本看板 |
| **项目经理** | 掌握项目预算执行情况 | 编制预算、管理进度节点、审批成本 |
| **成本会计** | 高效处理成本核算 | 审核成本入账、查看成本明细 |
| **项目成员** | 便捷提交工时和报销 | 填报工时、提交报销申请 |
| **审计人员** | 追溯成本发生轨迹 | 查询成本流水、审计轨迹 |

## 2. 核心业务流程

### 2.1 主流程（MVP闭环）

```
┌─────────────────────────────────────────────────────────────────┐
│                         MVP核心闭环                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    │
│   │ 项目立项 │───▶│ 预算编制 │───▶│ 成本填报 │───▶│ 审批入账 │    │
│   └─────────┘    └─────────┘    └─────────┘    └─────────┘    │
│        │                                    │                   │
│        │         ┌──────────────────────────┘                   │
│        │         ▼                                              │
│        │    ┌─────────┐    ┌─────────┐                        │
│        └───▶│ 预警监控 │◀───│ 成本看板 │                        │
│             └─────────┘    └─────────┘                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 各阶段详细说明

#### 阶段一：项目立项

- **触发条件**：项目经理发起项目立项申请
- **输入信息**：
  - 项目名称、项目编号
  - 关联客户信息
  - 预计开工/竣工日期
  - 预计合同金额
- **流程**：
  - 项目经理填写立项信息 → 提交企业主审批 → 审批通过后项目生效
- **输出**：状态为"已立项"的项目

#### 阶段二：预算编制

- **触发条件**：项目立项审批通过后
- **输入信息**：
  - WBS进度节点设置（节点名称、节点预算、节点产值）
  - 成本科目预算（人工费、材料费、设备费等科目）
- **流程**：
  - 项目经理编制预算 → 提交企业主审批 → 审批通过后预算生效
- **输出**：审批通过的预算方案

#### 阶段三：成本填报

- **触发条件**：项目开工后，项目成员填报成本
- **输入信息**：
  - **工时填报**：关联项目、工作类型、填报日期、工时数
  - **报销申请**：关联项目、成本科目、金额、费用说明、发票
- **流程**：
  - 项目成员填报 → 提交审批
- **输出**：待审批的工时/报销单

#### 阶段四：审批入账

- **触发条件**：成本单据提交后
- **处理角色**：项目经理/成本会计
- **流程**：
  - 审批人审核单据 → 通过后正式入账 → 预算余额扣减
- **输出**：已入账的成本记录

#### 阶段五：预警监控

- **触发条件**：成本入账时实时判断
- **预警规则**：
  - 黄色预警：预算使用达80%
  - 橙色预警：预算使用达90%
  - 红色预警：预算使用超100%
- **流程**：
  - 系统自动判断 → 触发预警 → 通知责任人
- **输出**：预警通知

#### 阶段六：成本看板

- **触发条件**：管理层查看
- **内容**：
  - 所有项目预算执行进度一览
  - 单项目预算-实际-产值对比
  - 已完工项目盈亏分析
- **输出**：可视化成本分析报表

## 3. 功能需求

### 3.1 功能模块总览

| 模块 | 功能点 | 角色 | 优先级 |
|------|--------|------|--------|
| **项目管理** | 项目立项、项目列表、项目详情 | 项目经理、企业主 | P0 |
| **预算管理** | WBS节点设置、预算编制、预算审批 | 项目经理、企业主 | P0 |
| **成本管理** | 工时填报、报销申请、成本审批入账 | 项目成员、项目经理、成本会计 | P0 |
| **监控预警** | 预算监控、三级预警、预警通知 | 系统自动触发 | P0 |
| **成本看板** | 项目总览、执行对比、盈亏分析 | 企业主、项目经理 | P0 |

### 3.2 项目管理模块

#### 3.2.1 项目立项

**功能描述**：项目经理发起项目立项申请，经企业主审批后项目生效。

**页面/表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目编号 | 文本 | 是 | 系统自动生成，格式：PRJ-YYYYMMDD-序号 |
| 项目名称 | 文本 | 是 | 项目全称 |
| 关联客户 | 下拉选择 | 是 | 从客户列表选择 |
| 预计开工日期 | 日期 | 是 | 格式：YYYY-MM-DD |
| 预计竣工日期 | 日期 | 是 | 必须晚于开工日期 |
| 预计合同金额 | 数字 | 是 | 单位：元，保留两位小数 |
| 项目简介 | 文本 | 否 | 最大500字 |

**流程说明**：
1. 项目经理填写立项信息
2. 提交审批，系统发送通知给企业主
3. 企业主审批（通过/驳回）
4. 审批通过后项目状态变更为"已立项"

**状态说明**：

| 状态 | 说明 |
|------|------|
| 草稿 | 立项信息未提交 |
| 审批中 | 等待企业主审批 |
| 已立项 | 审批通过，项目正式生效 |
| 已驳回 | 审批未通过，可修改后重新提交 |

#### 3.2.2 项目列表

**功能描述**：展示所有项目，支持按状态筛选。

**列表字段**：

| 字段 | 说明 |
|------|------|
| 项目编号 | 系统生成 |
| 项目名称 | 立项时填写 |
| 客户名称 | 关联客户 |
| 开工日期 | 预计开工日期 |
| 预算总额 | 审批通过的预算总额 |
| 执行进度 | 已执行预算/预算总额，百分比显示 |
| 项目状态 | 草稿/审批中/已立项/进行中/已完工 |

**操作**：

| 操作 | 可见角色 | 说明 |
|------|----------|------|
| 查看详情 | 全部 | 跳转项目详情页 |
| 编辑 | 项目经理（仅草稿状态） | 修改立项信息 |
| 提交审批 | 项目经理（仅草稿状态） | 提交立项审批 |
| 删除 | 项目经理（仅草稿状态） | 删除草稿项目 |

#### 3.2.3 项目详情

**功能描述**：展示单个项目的完整信息，包括预算、节点、成本明细。

**页面结构**：

```
项目详情
├── 基本信息
│   ├── 项目编号、项目名称
│   ├── 客户名称、合同金额
│   ├── 开工/竣工日期
│   └── 项目状态
├── WBS进度节点
│   ├── 节点列表（节点名称、预算、产值、完成进度）
│   └── 节点成本明细（点击展开）
├── 成本汇总
│   ├── 预算总额
│   ├── 已执行
│   ├── 预算余额
│   └── 执行率
└── 预警记录
    └── 当前预警状态
```

### 3.3 预算管理模块

#### 3.3.1 WBS进度节点设置

**功能描述**：项目经理设置项目的进度节点，每个节点可设置预算和产值。

**节点字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 节点编号 | 文本 | 是 | 自动生成，格式：ND-001 |
| 节点名称 | 文本 | 是 | 如：基础工程、主体结构 |
| 节点顺序 | 数字 | 是 | 用于排序，数字越小越靠前 |
| 节点预算 | 数字 | 是 | 单位：元 |
| 节点产值 | 数字 | 是 | 单位：元 |
| 预计完成日期 | 日期 | 是 | 节点计划完成日期 |

**业务规则**：
- 节点预算总和 = 项目总预算
- 节点产值总和 = 项目总产值（通常等于合同金额）
- 节点顺序必须唯一

#### 3.3.2 预算编制

**功能描述**：按成本科目编制项目预算，与WBS节点关联。

**预算字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 成本科目 | 下拉选择 | 是 | 从企业成本科目配置中选择 |
| 预算金额 | 数字 | 是 | 单位：元 |
| 关联节点 | 多选 | 是 | 关联的WBS节点 |
| 备注 | 文本 | 否 | 预算说明 |

**默认成本科目**（企业级配置）：
- 人工费
- 材料费
- 设备费
- 分包费
- 现场杂费
- 管理费
- 其他

**流程说明**：
1. 项目经理编制预算
2. 提交企业主审批
3. 审批通过后预算生效

**状态说明**：

| 状态 | 说明 |
|------|------|
| 草稿 | 预算未提交 |
| 审批中 | 等待企业主审批 |
| 已生效 | 审批通过，预算正式执行 |
| 已驳回 | 审批未通过 |

#### 3.3.3 预算审批

**功能描述**：企业主审批项目经理提交的预算方案。

**审批信息**：

| 字段 | 说明 |
|------|------|
| 申请人 | 项目经理姓名 |
| 申请时间 | 提交时间 |
| 项目名称 | 关联项目 |
| 预算总额 | 本次申请的预算总额 |
| 预算明细 | 按成本科目和节点的费用明细 |

**审批操作**：
- **通过**：预算状态变更为"已生效"
- **驳回**：填写驳回原因，预算退回项目经理修改

### 3.4 成本管理模块

#### 3.4.1 工时填报

**功能描述**：项目成员填报工时，系统自动计算人工成本。

**工时字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属项目 | 下拉选择 | 是 | 从已立项项目选择 |
| 工作类型 | 下拉选择 | 是 | 选项：设计、施工、监理、管理、会议、其他 |
| 关联节点 | 下拉选择 | 是 | 关联的WBS节点 |
| 填报日期 | 日期 | 是 | 格式：YYYY-MM-DD |
| 工时数 | 数字 | 是 | 单位：小时，0.5的倍数 |
| 工作内容 | 文本 | 是 | 本次工作描述，最大200字 |
| 附件 | 上传 | 否 | 支持图片、PDF |

**自动计算**：
- 工时单价值 = 项目人工费预算 / 项目总工时（由系统根据历史数据或配置计算）
- 本次工时成本 = 工时数 × 工时单价值

**流程**：
1. 项目成员填写工时表单
2. 提交审批（提交给项目经理）
3. 项目经理审批
4. 审批通过后正式入账

**状态说明**：

| 状态 | 说明 |
|------|------|
| 草稿 | 未提交 |
| 审批中 | 等待项目经理审批 |
| 已通过 | 审批通过，待入账 |
| 已入账 | 成本已计入项目 |
| 已驳回 | 审批未通过 |

#### 3.4.2 报销申请

**功能描述**：项目成员提交各类费用报销申请。

**报销字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属项目 | 下拉选择 | 是 | 从已立项项目选择 |
| 成本科目 | 下拉选择 | 是 | 从企业成本科目选择 |
| 关联节点 | 下拉选择 | 是 | 关联的WBS节点 |
| 费用类型 | 下拉选择 | 是 | 选项：差旅费、通讯费、交通费、招待费、办公费、其他 |
| 发生日期 | 日期 | 是 | 费用实际发生日期 |
| 报销金额 | 数字 | 是 | 单位：元，保留两位小数 |
| 费用说明 | 文本 | 是 | 费用描述，最大200字 |
| 发票张数 | 数字 | 是 | 发票数量，至少1张 |
| 发票附件 | 上传 | 是 | 发票照片或扫描件 |

**流程**：
1. 项目成员填写报销表单
2. 提交审批（提交给项目经理）
3. 项目经理审批
4. 审批通过后正式入账

**状态说明**：

| 状态 | 说明 |
|------|------|
| 草稿 | 未提交 |
| 审批中 | 等待项目经理审批 |
| 已通过 | 审批通过，待入账 |
| 已入账 | 成本已计入项目 |
| 已驳回 | 审批未通过 |

#### 3.4.3 成本审批入账

**功能描述**：项目经理/成本会计审批成本单据，审批通过后正式入账。

**待审批列表**：

| 字段 | 说明 |
|------|------|
| 单据编号 | 系统自动生成 |
| 单据类型 | 工时单/报销单 |
| 项目名称 | 关联项目 |
| 申请人 | 填报人 |
| 金额/工时 | 报销金额或工时数 |
| 提交时间 | 提交审批的时间 |
| 操作 | 查看详情、审批通过、驳回 |

**审批操作**：
- **通过**：单据状态变更为"已入账"，同时：
  - 预算余额扣减对应金额
  - 关联节点的已执行成本增加
  - 触发预算预警判断
- **驳回**：填写驳回原因，单据退回申请人

**成本入账后的数据更新**：
- 项目总预算余额 = 原余额 - 入账金额
- 对应节点的已执行成本 = 原已执行 + 入账金额
- 对应成本科目的已执行 = 原已执行 + 入账金额

### 3.5 监控预警模块

#### 3.5.1 预算监控

**功能描述**：实时展示各项目、各成本科目的预算执行进度。

**监控维度**：

| 维度 | 内容 |
|------|------|
| 项目级 | 预算总额、已执行、余额、执行率 |
| 节点级 | 节点预算、已执行、余额、执行率 |
| 科目级 | 科目预算、已执行、余额、执行率 |

**执行率计算**：
- 项目执行率 = 项目已执行成本 / 项目预算总额 × 100%
- 节点执行率 = 节点已执行成本 / 节点预算 × 100%
- 科目执行率 = 科目已执行成本 / 科目预算 × 100%

#### 3.5.2 三级预警规则

**功能描述**：当预算执行达到阈值时自动触发预警。

**预警级别**：

| 级别 | 触发条件 | 颜色标识 | 通知对象 |
|------|----------|----------|----------|
| 黄色预警 | 预算执行达80% | #FFA500 | 项目经理 |
| 橙色预警 | 预算执行达90% | #FF8C00 | 项目经理、企业主 |
| 红色预警 | 预算执行超100% | #FF0000 | 项目经理、企业主 |

**预警判断逻辑**：
```
IF (已执行成本 / 预算总额 × 100%) >= 100:
    预警级别 = 红色
ELIF (已执行成本 / 预算总额 × 100%) >= 90:
    预警级别 = 橙色
ELIF (已执行成本 / 预算总额 × 100%) >= 80:
    预警级别 = 黄色
ELSE:
    无预警
```

**预警触发时机**：
- 成本入账时实时判断
- 每日凌晨系统巡检

#### 3.5.3 预警通知

**功能描述**：预警触发后自动通知相关责任人。

**通知方式**：
- 系统内消息（实时推送）
- 站内信（保留历史记录）

**通知内容**：

| 内容 | 说明 |
|------|------|
| 预警项目 | 项目名称 |
| 预警级别 | 黄色/橙色/红色 |
| 预警维度 | 项目级/节点级/科目级 |
| 执行进度 | 当前执行率和金额 |
| 触发时间 | 预警发生时间 |

#### 3.5.4 预警记录

**功能描述**：查询历史预警记录。

**记录字段**：

| 字段 | 说明 |
|------|------|
| 预警编号 | 系统自动生成 |
| 项目名称 | 关联项目 |
| 预警级别 | 黄/橙/红色 |
| 预警维度 | 项目/节点/科目 |
| 执行进度 | 触发时的执行率 |
| 触发时间 | 预警发生时间 |
| 处理状态 | 待处理/已处理 |
| 处理时间 | 处理时间 |
| 处理人 | 处理人 |

### 3.6 成本看板模块

#### 3.6.1 项目总览看板

**功能描述**：一张表看清所有项目的预算执行情况。

**列表字段**：

| 字段 | 说明 |
|------|------|
| 项目编号 | 系统生成 |
| 项目名称 | 立项时填写 |
| 预算总额 | 审批通过的预算 |
| 已执行 | 已入账成本合计 |
| 预算余额 | 预算 - 已执行 |
| 执行率 | 已执行/预算 × 100% |
| 预警状态 | 无/黄/橙/红 |

**交互功能**：
- 按项目名称搜索
- 按预警状态筛选
- 点击项目名称跳转详情页

#### 3.6.2 预算执行对比

**功能描述**：展示单个项目的预算-实际-产值对比。

**展示形式**：分组柱状图

| 维度 | 说明 |
|------|------|
| 预算 | 计划投入 |
| 实际成本 | 已发生成本 |
| 产值 | 已确认产值（进度完成对应价值） |

**交互功能**：
- 支持按节点分组查看
- 支持按成本科目分组查看

#### 3.6.3 已完工项目盈亏分析

**功能描述**：分析已完工项目的盈利情况。

**分析维度**：

| 维度 | 说明 |
|------|------|
| 项目名称 | 完工项目 |
| 合同金额 | 项目总收入 |
| 实际成本 | 项目实际支出 |
| 利润 | 合同金额 - 实际成本 |
| 利润率 | 利润 / 合同金额 × 100% |

**列表字段**：

| 字段 | 说明 |
|------|------|
| 项目名称 | 完工项目名称 |
| 合同金额 | 项目总收入 |
| 实际成本 | 已入账成本合计 |
| 利润 | 合同金额 - 实际成本 |
| 利润率 | 利润占比 |
| 完工日期 | 项目完工日期 |

## 7. 术语表

| 术语 | 说明 |
|------|------|
| WBS | Work Breakdown Structure，工作分解结构，将项目分解为可管理的工作单元 |
| 预算余额 | 预算可用金额 = 预算总额 - 已执行金额 |
| 执行率 | 预算执行进度 = 已执行金额 / 预算总额 × 100% |
| 产值 | 项目或节点预计完成的工程价值 |
| 成本科目 | 成本费用的分类（如人工费、材料费、设备费等） |
| 入账 | 成本单据审批通过后正式计入项目成本 |
