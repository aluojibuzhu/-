## 1. 产品概述

### 1.1 产品定位

面向**中小型建筑企业**的**SaaS 工程项目成本管理平台**，解决企业项目成本管理中"预算执行失控"和"成本进度脱节"两大核心痛点。

### 1.2 MVP核心闭环

```
立项审批(含预算) → 成本填报 → 审批入账 → 预警监控 → 成本看板
       ↑                                                       │
       └───────────────────────────────────────────────────────┘
```

> **注意**：原始闭环中「预算编制」为独立阶段，现已合并入「立项审批」。项目经理在立项时同步完成WBS节点设置和成本科目预算分配。

### 1.3 目标用户与核心诉求

| 角色 | 核心诉求 | 在MVP中的职责 |
|------|----------|--------------|
| **超级管理员** | 系统最高权限 | 系统配置、角色权限管理、所有业务模块全部权限 |
| **管理员** | 掌控全局经营风险 | 审批项目立项、配置预警规则、查看全部成本数据 |
| **项目经理** | 掌握本项目预算执行情况 | 发起项目立项、管理WBS节点和预算、审批成本、查看本项目数据 |
| **成本会计** | 高效处理全局成本核算 | 审核成本入账、出具资金报表、查看全部项目成本数据 |
| **项目成员** | 便捷提交工时和报销 | 填报工时、提交报销申请、查看本人参与项目 |

### 2.3 与原始PRD的设计差异

实际实现过程中，项目立项模块做了以下重要调整，这些调整影响了后续模块的设计：

| 调整项 | 原始PRD | 实际实现 | 影响 |
|--------|---------|----------|------|
| 预算编制位置 | 立项后独立阶段，单独审批 | **合并入立项阶段**，WBS节点+成本科目预算与基本信息一同提交审批 | PRD 3.3「预算管理模块」核心内容已实现，无需单独开发 |
| 审批角色名称 | 企业主 | 管理员 | 角色映射调整 |
| 产值(Output Value) | 有产值概念 | MVP阶段移除 | 看板中"预算-实际-产值"对比简化为"预算-实际"对比 |
| 成本科目层级 | 仅一级7大类 | 支持二级科目，企业可配置 | 成本填报时科目选择更灵活 |
| 审批入口 | 各模块独立审批 | **统一收归模块3「审批入账」**，立项/工时/报销均从模块3入口审批 | 审批人一个入口处理所有待审批事项，模块1和模块2仅负责提交 |

### 3.2 各阶段详细说明

#### 阶段一：项目立项（含预算编制）

> **已实现**。实际实现将预算编制合并入立项阶段，项目经理在单页面完成完整方案。

- **触发条件**：项目经理发起项目立项申请
- **输入信息**：
  - 项目基本信息（项目名称、关联客户、开工/竣工日期、合同金额等）
  - WBS进度节点（节点名称、预计完成日期、顺序）
  - 成本科目预算分配（节点×科目矩阵，各科目金额手动填写，节点预算自动累计）
- **流程**：
  - 项目经理填写完整方案 → 保存草稿 → 提交审批 → 管理员在**模块3「审批入账」**中审批 → 通过后项目状态变为"已立项"
- **输出**：状态为"已立项"的项目（包含预算方案）

#### 阶段二：预算编制（已合并）

> **注意**：原始PRD中预算编制为独立阶段，现已合并入阶段一。此处保留原描述供参考。WBS节点设置和成本科目预算在立项时同步完成，无需单独审批。

#### 阶段三：成本填报

- **触发条件**：项目开工后，项目成员填报成本
- **输入信息**：
  - **工时填报**：关联项目、工作类型、填报日期、工时数
  - **报销申请**：关联项目、成本科目、金额、费用说明、发票
- **流程**：
  - 项目成员填报 → 提交审批 → 审批人在**模块3**中处理
- **输出**：待审批的工时/报销单

#### 阶段四：审批入账

> **设计更新（2026-04-30）**：模块1「项目立项审批」和模块2「工时/报销审批」的审批入口**统一收归至本模块**。模块3下分设**两个独立功能页面**：
> - **3.1 立项审批**：专门处理项目立项审批，管理员查看完整方案后通过/驳回
> - **3.2 成本审批**：统一处理工时单和报销单审批，审批通过后自动入账并触发预算扣减
> 
> 模块1和模块2仅负责「提交审批」，不内嵌审批操作UI。

- **触发条件**：立项/工时/报销单据提交后
- **处理角色**：管理员（立项审批）、项目经理（成本审批）
- **功能页面**：
  - **3.1 立项审批**：独立侧边栏菜单 → 待审批立项列表 → 查看完整方案（基本信息+WBS+预算矩阵）→ 通过/驳回
  - **3.2 成本审批**：独立侧边栏菜单 → 待审批成本列表（工时+报销，可切换）→ 查看详情 → 通过/驳回 → 通过后自动入账
- **入账处理**（3.2成本审批通过后自动触发）：
  - 预算余额扣减 → 节点已执行成本更新 → 科目已执行成本更新 → 触发预警判断
- **输出**：已入账的成本记录

#### 阶段五：预警监控

- **触发条件**：成本入账时实时判断
- **预警规则**：
  - 黄色预警：预算使用达80%
  - 橙色预警：预算使用达90%
  - 红色预警：预算使用超100%
- **流程**：
  - 系统自动判断 → 触发预警 → 通知责任人
- **输出**：预警通知

#### 阶段六：成本看板

- **触发条件**：管理层查看
- **内容**：
  - 所有项目预算执行进度一览
  - 单项目预算-实际-产值对比
  - 已完工项目盈亏分析
- **输出**：可视化成本分析报表

#### 4.3.1 WBS进度节点设置

**功能描述**：项目经理设置项目的进度节点，每个节点可设置预算和产值。

#### 4.4.3 成本审批入账

**功能描述**：项目经理/成本会计审批成本单据，审批通过后正式入账。

**待审批列表**：

| 字段 | 说明 |
|------|------|
| 单据编号 | 系统自动生成 |
| 单据类型 | 工时单/报销单 |
| 项目名称 | 关联项目 |
| 申请人 | 填报人 |
| 金额/工时 | 报销金额或工时数 |
| 提交时间 | 提交审批的时间 |
| 操作 | 查看详情、审批通过、驳回 |

**审批操作**：
- **通过**：单据状态变更为"已入账"，同时：
  - 预算余额扣减对应金额
  - 关联节点的已执行成本增加
  - 触发预算预警判断
- **驳回**：填写驳回原因，单据退回申请人

**成本入账后的数据更新**：
- 项目总预算余额 = 原余额 - 入账金额
- 对应节点的已执行成本 = 原已执行 + 入账金额
- 对应成本科目的已执行 = 原已执行 + 入账金额

### 4.5 监控预警模块

> **设计更新（2026-05-06）**：基于 brainstorming，模块4重构为 3 个独立功能页面（4.1 预警看板 + 4.2 预警中心 + 4.3 预警规则配置），详见 `docs/superpowers/specs/2026-05-06-alert-monitoring-design.md`。原PRD中4个子模块（预算监控/三级预警/预警通知/预警记录）已合并升级。

#### 4.5.1 预警看板（可视化仪表盘）

**功能描述**：项目经理视角的可视化仪表盘，全局感知预算异常。所有数据卡片和图表区域均可点击下钻，弹出详情弹窗。

**页面布局**：
- 预警概览统计（待处理/今日新增/本月累计/已处理）
- 项目健康度总览（正常/黄色/橙色/红色四色卡片）
- 预算消耗趋势图（折线：预算总额 vs 实际成本累计，近12个月）
- 成本科目消耗分布（分组柱状图 + 消耗占比圆环图）
- 项目预警Top10（横向条形图 + 同步表格）

**交互规则**：点击任意卡片/图表→弹出下钻弹窗（不离开看板），弹窗内可跳转项目详情或预警中心。

#### 4.5.2 预警中心

**功能描述**：预警记录的集中处理中心。从触发→通知→确认/忽略/跟进→关闭，完整处理闭环。

**预警列表**：红色→橙色→黄色默认排序，按级别/类型/状态/时间筛选。

**处理状态**：待处理(0)→已确认(1)/已忽略(2)→跟进中(3)→已关闭(4)

**详情抽屉**：点击预警编号右侧滑出，展示信息网格 + 预算对比 + 处理历史时间线 + 操作按钮。

#### 4.5.3 预警规则配置

**功能描述**：管理员灵活定义预警规则。不再写死 80/90/100 阈值。

**5种规则类型**：
| 类型 | 触发条件 | 说明 |
|------|----------|------|
| 预算执行率 | 执行率 >= X% | 项目/节点/科目超预算阈值 |
| 单笔大额入账 | 单笔金额 >= X元 | 异常大额支出 |
| 预算余额不足 | 余额 <= X% | 剩余预算过低 |
| 项目延期 | 超竣工日 >= X天 | 工期超期仍在花钱 |
| 静默无入账 | 连续 >= X月 | 项目可能停滞 |

**每条规则可配置**：触发条件+预警级别（黄/橙/红）+ 作用范围（全局/指定项目/指定科目）+ 通知角色（多选）+ 静默期（同规则N小时内不重复）+ 启用/停用开关。

**预置5条默认规则**：执行率红(≥100%)/橙(≥90%)/黄(≥80%) + 单笔大额(≥¥50,000) + 预算余额(≤10%)

#### 4.5.4 预警触发机制

**实时触发**：成本入账时同步检查（CostApprovalServiceImpl 注入）

**定时巡检**：Quartz 每日凌晨2点（覆盖延期/静默规则，入账事件无法触发的类型）

**通知方式**：复用若依 sys_notice + HeaderNotice 铃铛组件推送

**静默期**：同一项目同一规则在静默期内（如24小时）不重复触发通知

## 8. 术语表

| 术语 | 说明 |
|------|------|
| WBS | Work Breakdown Structure，工作分解结构，将项目分解为可管理的工作单元 |
| 预算余额 | 预算可用金额 = 预算总额 - 已执行金额 |
| 执行率 | 预算执行进度 = 已执行金额 / 预算总额 × 100% |
| 产值 | 项目或节点预计完成的工程价值 |
| 成本科目 | 成本费用的分类（如人工费、材料费、设备费等） |
| 入账 | 成本单据审批通过后正式计入项目成本 |
