# Commit: Project Establishment Frontend Matrix

## 提交主题

完善项目立项矩阵与基础配置菜单

## 背景

本次提交围绕项目管理 MVP 的项目立项模块收口，重点解决前端矩阵录入体验、项目详情页规范化、基础配置菜单不可见、登录后公告缺表报错等问题，并补齐成本科目基础配置的前后端维护能力。

## 主要改动

- 项目立项表单
  - 将 WBS 节点与成本科目预算矩阵改为更适合表格内编辑的数字输入。
  - 支持节点上移、下移和删除。
  - 提交审批前自动保存脏数据，避免未保存修改丢失。
  - 统一表格列居中，符合前端设计规范。

- 项目详情页
  - 注册静态隐藏详情路由，修复从项目列表进入详情时 404 的问题。
  - 补充加载态和参数保护，避免 `projId=undefined` 触发后端类型转换错误。
  - 按详情页规范重构基本信息、汇总卡片和 WBS 成本矩阵。
  - 修复普通字段标签误换行问题，仅长文本字段允许自然换行。

- 基础配置
  - 新增成本科目维护页面，支持查询、新增、编辑、删除和二级科目树。
  - 补齐成本科目后端 CRUD、服务校验和 MyBatis 映射。
  - 修复菜单数据，将 PRD 1.4「基础配置」挂到「项目管理」下。
  - 隐藏「项目详情」侧边栏菜单，仅保留列表页跳转入口。

- 登录与公告
  - 登录页永久省略验证码输入。
  - 后端在验证码参数为空时跳过验证码校验，避免影响其他登录流程。
  - 新增公告表增量 SQL，修复登录后顶部通知查询 `sys_notice` 缺表错误。

- 文档与 SQL
  - 更新 PRD MVP 功能树，明确项目管理 1.1-1.4 已完成。
  - 更新前端设计规范中表格列对齐和表格内数字输入规则。
  - 新增项目启动说明文档。
  - 新增基础配置菜单和公告表增量 SQL。

## 注意事项

- 若前端侧边栏仍看不到「基础配置」，需要退出登录后重新登录，让若依重新拉取动态路由。
- PowerShell 直接管道导入含中文 SQL 可能导致中文乱码，建议使用 MySQL 客户端工具导入 SQL 文件，或确保以 UTF-8 编码执行。
