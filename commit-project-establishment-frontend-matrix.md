# Commit 文档：项目立项前端规范化与 WBS 成本矩阵

## 背景

本次修改基于 `claude-review.md` 的复审意见，以及 `docs/superpowers/specs/2026-04-29-frontend-specification-design.md` 中新增的前端规范。

提交信息：`feat: implement WBS cost matrix UI`

提交哈希：以 `git log -1 --oneline` 输出为准。

## 本次解决

- 实现 WBS 节点 × 成本科目矩阵表：
  - 成本科目从后端动态加载。
  - 每个 WBS 节点按一级成本科目填写分配金额。
  - 节点预算改为自动汇总，前端不可手动编辑。
  - 保存时自动构建 `allocations` 数据提交给后端。
- 详情页展示 WBS 成本分配矩阵，能够查看每个节点各成本科目的分配金额。
- 提取 `src/utils/project.js`，集中维护项目状态映射、状态标签类型、金额格式化和可编辑状态判断。
- 按前端规范调整：
  - 侧边栏主题改为浅色。
  - 列表、表单、详情页补充 `name`、统一 API 调用风格和页面结构。
  - 表单页增加离开未保存提示。
  - 表单页增加预计开工/竣工日期交叉校验。
- 后端增强：
  - 合同金额必须大于 0。
  - WBS 节点名称、预计完成日期和节点预算合法性校验。
  - 保存草稿时将前端临时节点 ID 映射到数据库生成的真实节点 ID，保证成本分配记录能正确关联 WBS 节点。
  - 客户名称变更后同步项目表冗余客户名称。

## 判断后暂缓

- `saveDraft` 深度拆分：当前方法仍偏大，但矩阵落地后主流程清晰，后续进入预算版本或独立节点编辑时再拆分更合适。
- 审批业务角色额外约束：当前设计文档以若依权限 `project:projInfo:approve` 控制管理员审批，暂不叠加“不能审批自己项目”的业务规则。
- DTO 全量分离和审批事件机制：属于架构增强项，不影响当前立项闭环。
- 成本科目树管理页面：本次只满足立项矩阵动态列需求，管理页留给后续系统配置迭代。
